## 架构约束（不可违反）

1. **改源不改衍生**：所有功能改动先在 `单词闯关开发版.html` 完成，再用 `tools/build-*.js` 重新生成衍生版。绝不手改 `单词闯关四年级通关版.html` 或 `单词闯关本地版.html`。
2. **build 脚本断言唯一命中**：每处字符串替换必须唯一匹配，不命中即抛错（防静默失败）。改了开发版 HTML 后，若 build 脚本的 `old` 串不再匹配，需同步更新 build 脚本。
3. **改后必校验**：改开发版内联 JS 后跑 `node -e` 语法检查；改完衍生版跑 `node tools/verify-share.js`（44 项断言）；涉及交互逻辑用 jsdom 真实 DOM 模拟验证。
4. **三版本 LS_KEY 隔离**：开发版 `.v1`、四年级版 `.share.v1`、本地版 `.local.v1`，互不干扰。
